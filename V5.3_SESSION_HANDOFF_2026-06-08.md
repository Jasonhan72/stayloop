## 3. HOW TO DEPLOY (critical — read carefully)

Production = **Cloudflare Pages**, deployed via `wrangler pages deploy ... --branch main`.
The git remote is `github.com/Jasonhan72/stayloop.git`; work is pushed to the
**`v5.3-launch`** branch (the local working branch is `v5`).

### The deploy script (lives in repo root, double-click in Finder)
`ship2-v53.command` does: commit+push → `npx @cloudflare/next-on-pages@1` build →
deploy with **global `wrangler`** (NOT `npx wrangler`).

```bash
#!/bin/bash
cd "$(dirname "$0")" || exit 1
exec > >(tee ship2.log) 2>&1
rm -f .git/HEAD.lock .git/index.lock
git add -A
git commit -m "feat(v5.3): ..." || echo "(nothing to commit)"
git push origin HEAD:v5.3-launch || echo "(push skipped)"
export NODE_OPTIONS=--max-old-space-size=4096
npx @cloudflare/next-on-pages@1
[ "$?" != "0" ] && { echo "BUILD FAILED — NOT DEPLOYING"; exit 1; }
wrangler pages deploy .vercel/output/static --project-name stayloop --branch main --commit-dirty=true
echo "SHIP2 DONE"
```

### GOTCHAS (learned the hard way)
- **`npx wrangler` HANGS** (waits on an install prompt). Use the **globally
  installed `wrangler`** (the user's `wrangler 4.64.0`) for the deploy step.
  `npx @cloudflare/next-on-pages` for the build step is fine.
- The first cold `next build` took ~60 min on this Mac; warm rebuilds are ~3–4s
  (the `.next/cache` makes incremental builds fast).
- A `.command` file must be `chmod +x`. Files created by the assistant's editor
  are local (no quarantine) so they double-click fine.
- In Finder, if clicks don't register on a `.command`, select it with arrow keys
  and press `Cmd+O` to open/run it.

### Why the assistant can't deploy from the sandbox
The Cowork sandbox is isolated from the Mac's git credentials AND can't reliably
read freshly-written mounted files. So the assistant: writes code with the file
tools → drops a `.command` script → uses computer-use (Finder) to run it on the
Mac → watches the Terminal output via screenshots → verifies with `web_fetch` of
the live URL (cache-bust with `?v=<ts>`).
